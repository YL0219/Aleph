# Project Aleph: Phase 4 Roadmap & Architecture

## Core Architecture: The Cybernetic Triad
We have evolved into a rigid, Domain-Driven Design (DDD) named **Project Aleph**. The system mimics a biological organism relying on three sovereign sectors:

1. **Axiom (The Body / Senses):** Handles databases, web scrapers, data ingestion pipelines, API controllers, and physical trading execution.
2. **Arbiter (The Conscious Mind):** Handles cognitive routing, LLM agents, narrative analysis, and playbook execution.
3. **Aether (The Autonomic Nervous System):** The dark-box environment for Bayesian math, macroeconomic probability matrices, and system rhythm.

### Pillar 3: The Quant Engine & Homeostasis [🔄 IN PROGRESS]
* **Quant Logic [Done]:** Python environment wired for Pandas-TA indicator scoring and Rule-based Macro Regime checks.
* **The Master Loop [Current Focus]:** Strip multiple competing background timers. `HeartbeatService` inside Aether becomes the *Single Master Loop* (The Heart). It explicitly triggers Axiom's ingestion cycle before running math.
* **Homeostasis [Current Focus]:** Build `IHomeostasis` inside Aether to act as the autonomic regulator. 
    * Tracks *External Stress* (Adrenaline/Market Danger) injected by Arbiter/Axiom.
    * Tracks *Internal Fatigue* (Overload/Breathlessness) generated by Heartbeat.
* **The Governor [Current Focus]:** Heartbeat calculates safe BPM based on Homeostasis state. Degrades workload if the system is breathless.
* **Circulatory Foundation [Current Focus]:** Establish a basic `PulseEnvelope` or `SystemSignal` C# record to standardize how Heartbeat and Homeostasis share data (laying groundwork for a future Event Bus).

## Execution Policies
* **Python Environment:** Backend MUST run Python using `PythonPathResolver`.
* **Process Calls:** MUST use `ProcessRunner.RunAsync` with `ArgumentList`. No string concatenated arguments.
* **Observability:** All logging via `ILogger<T>` (Serilog). Standard prefixes: `[Axiom]`, `[Arbiter]`, `[Aether]`, `[Python]`, `[Heartbeat]`.
